3. How to edit this homepages¶

このページは、git及びgithubを使って管理されているため、誰でも編集することができます。

gitはオリジナルを編集せずに自分だけで編集ができ、後で統合することができるためとても便利です。

ここではgitの使い方及び編集の仕方を解説します。

3.1. アカウント作成¶

gitのアカウントとgithubのアカウントは別物です。

まず、git入門を参考に、gitのアカウントを作成します。

次に、githubアカウントを参考に、アカウントを作ります。

更にgithub sshを参考にアカウントに公開鍵を登録しておくと便利です。（既存のものでいい）

3.2. 編集の流れ¶

図で示すと、

上記のようになっており、皆さんには左の開発者の1~5を行ってもらいます。

3.2.1. リモートリポジトリをpull（上図１）¶

わかりやすく言うと、ネットワーク上にあるファイルを持ってくるだけです。

今回は、~/Document/webで作業しますが、実際はどこでも構いません。

cd ~/Document/web/ #ディレクトリの移動
git clone https://github.com/tanaty5828/varidato_source #データのダウンロード

3.2.2. ブランチを分ける（上図２）¶

このままではオリジナルが変更されてしまうので、ブランチを分けます。

わかりやすく言うと、変更するからオリジナルを残してね〜という意味になります。

cd varidato_source #ディレクトリの移動
git branch ishikawa #ブランチの作成、名前は自由です。
git checkout ishikawa #ブランチの移動

3.2.3. 編集をする（上図３）¶

varidato_source/の中身は、内容ごとにディレクトリがわけられています。

編集のテスト用にMD/というディレクトリがあるので(現在はない)これに入り、

中にあるgromacs.rstを編集してみましょう。

cd MD
vi gromacs.rst
#適当に編集…

3.2.4. pushする（上図４）¶

ここまでの編集をネットに送ります。

cd ~/Document/web/varidato_source/ #varidato_source/まで戻る。
git add . #変更履歴の保存
git commit -m "test" #どのような変更を行ったかのメモ
git push https://github.com/tanaty5828/varidato_source hishikawa #送る。"hishikawa"はブランチを分けたときの名前にする。
## githubのUsernameとpasswordを入力

3.2.5. pullリクエストの作成（上図５）¶

このままだと変更は管理者に伝わってないので、プルリクエストを送ります。

プルリクエストの作成を参考に、送っていただければすぐに反映します。

3.2.6. ローカルにあるものを消す。¶

push が一通り終わったら作業したものは消して、また編集したいときには

git clone https://github.com/tanaty5828/varidato_source #データのダウンロード

から始めるといいです。

3.3. テキストの記載法¶

ソースコードの書き方には、2種類あり、

  • Markdown(.md)

  • reStructuredText(.rst)

があります。Markdownのほうが書きやすいけどできることは少なく、reStructuredTextのほうが書きにくく高機能です。

詳しい書き方は、

  • Markdown

  • reStructuredText

を参考にしてください。

エディターはvimでもemacsでもvscodeでも何でもいいですが、Markdownを使うならMacでもWindowsでも使えるTypora がおすすめです。

3.5. Git の便利なコマンドまとめ¶

3.5.1. 現在の状況を表示する¶

# 現在の状況を表示
git status

# output

# On branch master
# Changes not staged for commit:
#   (use "git add <file>..." to update what will be committed)
#   (use "git checkout -- <file>..." to discard changes in working directory)
#

#

3.5.3. ファイルを編集したが、やっぱりgit cloneしたときの状態に戻したい(まだgit add .していない。)¶

# myfile.txt を元の状態に戻したい。
git checkout HEAD myfile.txt

# ブランチ全体を戻す。
git checkout HEAD

上のコマンドを推奨する。

3.5.4. ファイル内の変更箇所を見たい¶

# myfile.txtの差分を表示
git diff myfile.txt

# output

-# output
+# output_data

削除した行は-で、追加した行は+で表示される。

3.5.5. ブランチの表示¶

#すべてのブランチ（リモート・ローカル問わず）を表示
git branch -a

# output
  origin/Gnuplot
  origin/master

3.5.6. ブランチの作成、移動をまとめて¶

# ブランチの作成、移動を１コマンドで
git checkout -b ishikawa

# こうするより楽
git branch ishikawa
git checkout ishikawa
